# Meditation User Story


Exercise 2: Login/Registration Page
Title:
As a user, I want to register by entering my username, email, and password so that I can create an account.

Acceptance Criteria:

Users can enter valid details and click “Sign Up” to create an account.

An error message is shown if any input is invalid or missing.

Registration is successful if all inputs are valid.

Story Points: 3

Title:
As a user, I want to log in using my email and password so that I can access my account.

Acceptance Criteria:

Users can log in with correct credentials and are redirected to their dashboard.

An error message is displayed for incorrect credentials.

Login is prevented if fields are empty or invalid.

Story Points: 3

Title:
As a user, I want to receive feedback when I attempt to sign up or log in without entering details so that I can fix the errors.

Acceptance Criteria:

Error messages are displayed for missing fields on sign-up or login attempts.

Each error clearly identifies the missing or incorrect field.

The user is prompted to complete the form before submission.

Story Points: 2

Title:
As a user, I want my details to be stored in local storage so that my data persists between sessions.

Acceptance Criteria:

User details are saved in local storage after registration.

Login uses stored credentials for validation.

Data remains available even after app restart.

Story Points: 3

Exercise 3: Homepage
Title:
As a user, I want a personalized greeting with my name and a title so that I feel welcomed and encouraged to meditate.

Acceptance Criteria:

Display “Hello, [username]” at the top.

Follow with the subtitle “Find your perfect meditation.”

Personalization appears immediately after login.

Story Points: 2

Title:
As a user, I want to see popular meditation cards so that I can explore options based on my preferences.

Acceptance Criteria:

Display meditation cards with images and titles.

Cards include categories such as calmness, relaxation, and durations.

Cards are responsive and scrollable.

Story Points: 3

Title:
As a user, I want a daily featured meditation so that I can quickly access a recommended session.

Acceptance Criteria:

Highlight one meditation in a dedicated section.

Include image, title, category, and duration.

Section updates daily.

Story Points: 2

Title:
As a user, I want intuitive navigation icons so that I can easily move around the app.

Acceptance Criteria:

Display app logo in the top-left.

Display a settings icon in the top-right.

Tapping icons navigates to respective screens.

Story Points: 2

Exercise 4: Detailed Screen
Title:
As a user, I want an “About” section for each exercise so that I can understand its benefits and purpose.

Acceptance Criteria:

Provide a brief and clear description.

Explain the mental/physical benefits.

Section appears below the exercise title.

Story Points: 2

Title:
As a user, I want an “Instructions” section for each exercise so that I can perform it correctly.

Acceptance Criteria:

Step-by-step guidance is provided.

Instructions are concise and readable.

Section includes tips on posture and breathing.

Story Points: 3

Title:
As a user, I want an “Add to Favorites” button so that I can easily save an exercise for future practice.

Acceptance Criteria:

Button appears at the bottom of the screen.

Button toggles between “Add” and “Remove.”

Heart icon updates with status.

Story Points: 2

Title:
As a user, I want navigation icons for sharing and going back so that I can easily manage the exercise page.

Acceptance Criteria:

A back icon appears at the top-left.

A share icon appears at the top-right.

Tapping icons performs the respective action.

Story Points: 2

Exercise 5: Add to Favorites Functionality
Title:
As a user, I want to add an item to my Favorites so that I can save activities or articles I like for quick access later.

Acceptance Criteria:

A heart icon and “Add to Favorites” button are visible.

Tapping adds the item and updates the icon.

Button text changes to “Remove from Favorites.”

Story Points: 3

Title:
As a user, I want to remove an item from my Favorites so that I can manage my saved content.

Acceptance Criteria:

Button displays “Remove from Favorites” when active.

Tapping removes the item and reverts the icon.

Users can toggle this state anytime.

Story Points: 2

Title:
As a user, I want a “My Favorites” screen so that I can view and manage all my saved items in one place.

Acceptance Criteria:

Saved items are listed with title, category, and duration.

Each item is clickable to view or start.

List is organized and scrollable.

Story Points: 3

Exercise 6: Daily Reminders
Title:
As a user, I want to view the calendar for the current month and navigate between months so that I can easily select dates for reminders.

Acceptance Criteria:

Display current month and days.

Provide navigation arrows.

Selection highlights chosen date.

Story Points: 3

Title:
As a user, I want to select a date and time for a reminder so that I can schedule it properly.

Acceptance Criteria:

Display default text for date/time status.

Allow selection via UI controls.

Show chosen date and time upon selection.

Story Points: 3

Title:
As a user, I want to add a reminder after selecting a time so that I can schedule it for a future date and time.

Acceptance Criteria:

Provide “Add Reminder” button.

Button is only active when time is selected.

Reminder is saved and listed.

Story Points: 2

Title:
As a user, I want to see a list of all my reminders so that I can manage them easily.

Acceptance Criteria:

Show all scheduled reminders with date and time.

Each has a red “Delete” button.

Deletion removes reminder from list.

Story Points: 3

Exercise 7: Sharing Exercises
Title:
As a user, I want to easily share recommended exercises with friends or family so that I can help others discover helpful activities.

Acceptance Criteria:

Display share icon on detail page.

Allow sharing via multiple apps or platforms.

Share content includes exercise name and link.

Story Points: 2

Exercise 8: Logout Functionality
Title:
As a user, I want a clear and visible logout button so that I can easily log out of my account when I’m done using the app.

Acceptance Criteria:

“Logout” button is clearly labeled.

Tapping logs the user out and redirects to login.

Session data is cleared securely.

Story Points: 2

Exercise 9: Change Settings
Title:
As a user, I want to switch between light and dark themes so that I can reduce eye strain and customize the app’s visual experience.

Acceptance Criteria:

Provide theme toggle in settings.

Theme updates immediately without refresh.

Theme choice persists across sessions.

Story Points: 2
